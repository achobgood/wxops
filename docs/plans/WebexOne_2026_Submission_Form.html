Section 01

Abstract

CUCM end-of-life is the most urgent topic in enterprise collaboration, but most migration sessions stop at slides and architecture diagrams. This instructor-led lab goes further: attendees build, migrate, and operate a Webex Calling environment themselves — start to finish — by working with an AI agent that understands the full Webex Calling API surface and handles the execution complexity for them.

The platform pairs a purpose-built automation layer covering 828 Webex API endpoints with 16 domain-specific AI skills. Attendees describe what they want — "create a location with PSTN," "build an auto attendant with these menu options," "migrate this CUCM cluster" — and the AI agent designs the approach, executes the API calls, handles dependencies and edge cases, and verifies the results. The attendee makes the decisions. The AI handles the mechanics.

The lab follows a three-act structure. In Act 1, attendees deploy a complete green-field Webex Calling environment from scratch — locations, users, auto attendants, call queues, hunt groups, dial plans, and call routing — then verify by calling the main number and hearing the auto attendant answer. In Act 2, attendees execute a full CUCM-to-Webex migration against a live CUCM 15.x cluster with 80 pre-provisioned objects across 21 types. They extract via AXL, run analysis pipelines that decompose CSSes and flag conflicts, make decisions on ambiguous items, provision into Webex, and verify migrated call flows work. In Act 3, attendees shift to day-2 operations: bulk onboarding, CDR reporting, compliance audits, and debugging a live misconfiguration.

Attendees leave with the platform installed, a working environment they built themselves, and a repeatable AI-assisted migration methodology they can apply to customer engagements on Monday morning. No prior API, CLI, or coding experience is required — just basic Webex Calling familiarity. The AI agent is the interface.

Built by a Cisco SE who got tired of manual, error-prone deployments and decided to fix the problem with code — then made it accessible to anyone who can describe what they want in plain English.

Section 02

Detailed Description

Enterprise collaboration teams face a defining moment. CUCM end-of-life timelines are accelerating, and every partner and IT organization needs to answer the same question: how do we get from on-premises to cloud calling — reliably, repeatably, and without rebuilding everything by hand in Control Hub?

This four-hour instructor-led lab answers that question with a new interaction model: attendees tell an AI agent what they want to build, and the agent handles the API complexity.

The Platform. The lab is built on an AI-guided operations platform that combines two layers. The automation layer exposes 828 Webex API endpoints across 100 command groups, covering the full Webex Calling surface: provisioning, call features, person and location settings, call routing, device management, SCIM identity, licensing, audit/compliance, messaging, and real-time call control. The intelligence layer adds 16 domain-specific AI skills — each encoding the operational knowledge, dependency ordering, edge cases, and verification steps required to complete a category of work correctly.

Attendees interact through conversation, not command syntax. They describe what they need, and the AI agent selects the appropriate skill, plans the execution sequence, runs the API calls, handles errors and edge cases, and verifies the results. The attendee focuses on what to build and why. The AI handles how.

This is not a product demo. It is a working tool built by a practitioner solving real deployment problems — tested against production Webex environments and a live CUCM 15.x cluster with 80 provisioned objects across 21 types.

Act 1 — Green-Field Deployment (90 minutes). Attendees start with an empty Webex Calling org and build a complete environment from scratch. Working with the AI agent, they create locations with PSTN connectivity, license and assign users, build auto attendants with menu trees, configure call queues and hunt groups, set up dial plans with translation patterns, and wire everything into working call flows. The agent manages the dependency chain — locations before users, users before features, features before routing — so attendees focus on design decisions, not execution order. The checkpoint is concrete: call the main number, hear the auto attendant, get routed through the call flow attendees just built.

Act 2 — CUCM Migration (75 minutes). Attendees connect to a lab CUCM cluster pre-loaded with 80 objects across 21 types. The AI agent extracts everything via AXL, then runs the analysis pipeline that converts CUCM dial plan patterns to Webex format, decomposes multi-partition CSSes into Webex-native routing structures, maps device models to supported hardware, and detects conflicts. The pipeline produces decision reports separating what auto-resolved from what needs human judgment. This is where the attendee's expertise matters most: reviewing flagged items and making the calls that determine migration quality. The AI explains each conflict, presents options, and waits for the human decision. Then the agent executes the migration plan, provisioning into the sandbox org in dependency order and verifying the result end to end.

Act 3 — Day-2 Operations (30 minutes). The same platform that built and migrated the environment handles ongoing operations. Attendees bulk-onboard a new department, pull CDR reports and license utilization data, run a compliance audit, and debug a live call flow issue introduced by the instructor. The point is direct: this is not a migration tool you shelve after cutover — it is an operations platform for the life of the deployment.

Every step happens in the attendee's own session. There are no pre-built environments to click through and no slides to watch while an instructor demos. Attendees describe, decide, build, verify — and take the tools home.

Section 03

Learning Objectives

  1. Deploy a complete Webex Calling environment from scratch — locations, users, call features, and call routing — by directing an AI agent that handles API execution, dependency ordering, and verification.
  2. Execute a CUCM-to-Webex Calling migration end-to-end: AXL extraction, automated analysis with conflict detection, human-decision workflow, AI-orchestrated provisioning, and verification.
  3. Analyze CUCM migration decision reports to resolve ambiguous dial plan patterns, unsupported devices, and multi-partition CSS decomposition — the real-world judgment calls that determine migration success and that AI cannot make for you.
  4. Operate a production Webex Calling environment post-migration: bulk onboarding, CDR reporting, compliance auditing, and call flow troubleshooting using the same AI-guided platform.
  5. Apply a repeatable, AI-assisted migration methodology to your own customer engagements or enterprise deployments immediately after the session.
